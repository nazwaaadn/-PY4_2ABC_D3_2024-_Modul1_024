## Entri 5 — Singleton Pattern pada MongoService

| Komponen | Isian Mahasiswa |
|---|---|

| **Jawaban AI (Intisari)** | AI menyarankan pakai pola `factory` constructor dengan `static final _instance` dan private constructor `_internal()`. Setiap kali `MongoService()` dipanggil, selalu kembalikan instance yang sama. |
| **The Fact Check** | Saya tes dengan buat dua variabel `MongoService()` di tempat berbeda, lalu cek dengan `identical()`. Hasilnya `true` — berarti Singleton berhasil, keduanya menunjuk objek yang sama. |
| **The Twist (Modifikasi)** | Saya tambahkan pengecekan `_db?.isConnected` sebelum setiap operasi CRUD dengan fungsi `_getSafeCollection()`. Jadi kalau koneksi putus di tengah jalan, otomatis reconnect tanpa crash. |

**Implementasi di `mongo_service.dart`:**
```dart
class MongoService {
  static final MongoService _instance = MongoService._internal();
  factory MongoService() => _instance;
  MongoService._internal();

  Future<DbCollection> _getSafeCollection() async {
    if (_db == null || !_db!.isConnected || _collection == null) {
      await connect();
    }
    return _collection!;
  }
}
```

## Entri 7 — RBAC (Role-Based Access Control)

| Komponen | Isian Mahasiswa |
|---|---|
| **Pertanyaan (Prompt)** | "Gimana cara bikin sistem izin di Flutter supaya user dengan role berbeda punya akses yang berbeda pula?" |
| **Jawaban AI (Intisari)** | AI menyarankan bikin `Map<String, List<String>>` sebagai matrix perizinan: key adalah nama role, value adalah list aksi yang diizinkan. Lalu buat fungsi `canPerform(role, action)` untuk mengeceknya. |
| **The Fact Check** | Saya test dengan role `"Anggota"` mencoba aksi `"delete"` — hasilnya `false` seperti yang diharapkan. Tapi ada kasus edge: gimana kalau Anggota mau edit/hapus *data miliknya sendiri*? |
| **The Twist (Modifikasi)** | Saya tambahkan parameter `isOwner` di fungsi `canPerform()`. Kalau role adalah `Anggota` dan dia pemilik data tersebut (`isOwner == true`), maka aksi `update` dan `delete` tetap diizinkan. Ini yang disebut *Owner-based RBAC*. |

**Implementasi di `access_control_service.dart`:**
```dart
static bool canPerform(String role, String action, {bool isOwner = false}) {
  // Anggota boleh update/delete data MILIKNYA sendiri
  if (role == 'Anggota' &&
      (action == actionUpdate || action == actionDelete)) {
    return isOwner;
  }
  final permissions = _rolePermissions[role] ?? [];
  return permissions.contains(action);
}
```

## Entri 9 — DateHelper: Relative Time dalam Bahasa Indonesia

| Komponen | Isian Mahasiswa |
|---|---|
| **Pertanyaan (Prompt)** | "Gimana cara nampilin waktu dalam format relatif kayak '5 menit yang lalu' di Flutter?" |
| **Jawaban AI (Intisari)** | AI menyarankan hitung selisih waktu dengan `DateTime.now().difference(date)`, lalu gunakan kondisi bertingkat untuk menentukan format yang tepat (detik, menit, jam, hari). |
| **The Fact Check** | Saya test dengan tanggal sangat lama (lebih dari 7 hari), dan AI lupa handle kasus itu — hasilnya hanya "X hari yang lalu" sampai ratusan hari. Perlu fallback ke format tanggal penuh. |
| **The Twist (Modifikasi)** | Saya pakai `intl` package dengan locale `id_ID` untuk format tanggal penuh (contoh: `"d MMM yyyy"`). Jadi kalau lebih dari 7 hari, tampil sebagai `"9 Mar 2026"` bukan `"X hari yang lalu"`. |

**Implementasi di `date_helper.dart`:**
```dart
static String relativeTime(DateTime date) {
  final diff = DateTime.now().difference(date);
  if (diff.inSeconds < 60) return "${diff.inSeconds} detik yang lalu";
  if (diff.inMinutes < 60) return "${diff.inMinutes} menit yang lalu";
  if (diff.inHours < 24) return "${diff.inHours} jam yang lalu";
  if (diff.inDays < 7) return "${diff.inDays} hari yang lalu";
  return DateFormat("d MMM yyyy", "id_ID").format(date); // Fallback
}
```

| **The Twist (Modifikasi)** | Saya ganti pengecekan dengan `results.any((r) => r != ConnectivityResult.none)` untuk men-support multi-network (WiFi + seluler aktif bersamaan). Saya juga pisahkan fungsi `startConnectivityWatch(teamId)` dan `cancelWatch()` supaya lifecycle-nya lebih bersih dari `LogView`. |

**Implementasi di `log_controller.dart`:**

```dart
void startConnectivityWatch(String teamId) {
  _connectivitySub = Connectivity().onConnectivityChanged.listen((results) async {
    final isOnline = results.any((r) => r != ConnectivityResult.none);
    isOnlineNotifier.value = isOnline;
    if (isOnline) await _syncPending(teamId);
  });
}
```
